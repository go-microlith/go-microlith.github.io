Services

The origin API implements its business logic by making calls on services in the domain. It uses the results of those calls to create a response for the user.

Let’s refactor our domain into services which perform the work of creating words. Then we’ll refactor our origin endpoint to use those services to build our “Hello, world!” response.

The “word” Service

We will create a “word” service that returns a fixed word. It will have one endpoint which returns the word as a string and a public method that calls the endpoint.

At the code level, a service is an object which implements the tld.ServiceBuilder interface. The Build() method is called when the service is registered with a domain. It is passed a builder to define the service.

type Word struct {
  word     string
  client   *rest.Client
  endpoint *rest.Endpoint
}

func NewWord(word string) *Word {
  return &Word{
    word: word,
  }
}

func (word *Word) Build(service *tld.Service) {
  // TODO
}

Endpoint

We will add an endpoint to the service that returns the word. This largely mirrors adding an endpoint to the origin: we will create a Lambda handler and then wire the endpoint using the builder.

Lambda

The handler Lambda function for the endpoint will be a private instance method on the service object. It will simply return a 200 OK with the word as JSON.

func (word *Word) handler(ctx context.Context, req events.APIGatewayProxyRequest) (events.APIGatewayProxyResponse, error) {
  return rest.Respond(http.StatusOK, word.word)
}

Builder

Use the Scope() method on the service builder to add an endpoint. This method behaves just like the microlith.Scope() function in the origin.

We will capture the returned endpoint value in an instance variable so we can use it to make calls on the “word” endpoint.

func (word *Word) Build(service *tld.Service) {
  service.Scope("/", func(scope *rest.Scope) {
    word.endpoint = scope.Get("word", word.handler)
  })
}

Client

We will implement a public instance method that call the “word” endpoint and return the response body. To make calls on the service, we need to create a REST client using the builder.

Builder

Use the tld.ServiceClient() function create a rest.Client in your service builder. We will use this client to make calls against the service.

func (word *Word) Build(service *tld.Service) {
  // ... Same as before ...

  var err error
  word.client, err = tld.ServiceClient(service)
  if err != nil {
    log.Panic(err)
  }
}

Client

Use the client created in the builder to Call() endpoints. The returned value is a standard http.Response. Microlith provides the ResponseBody() and ErrorResponse() helpers to deserialize the contents of the response body either as an object or an error, respectively.

func (word *Word) Word(ctx context.Context) (string, error) {
  resp, err := word.client.Call(ctx, word.endpoint, nil)
  if err != nil {
    return "", err
  }

  switch resp.StatusCode {
  case http.StatusOK:
    msg := ""
    if err := rest.ResponseBody(resp, &msg); err != nil {
      return "", err
    }
    return msg, nil
  default:
    err := rest.ErrorResponse(resp, nil)
    return "", err
  }
}

Done

We can have a “word” service we can use in our root domain to factor out the creation of our words.

Origin

We need to update the origin to create and use the “word” service to factor into services.

We will create two instances of the service, one returning “hello” and the other “world”. Then, we will call those from our origin handler to construct the “Hello, world!” response for the user.

Services

First, you must create instances of the services and register them with your domain using the microlith.Service() function.

var (
  helloSvc = NewWord("hello")
  worldSvc = NewWord("world")
)

func init() {
  // ... Same as before ...

  microlith.Service("hello", helloSvc)
  microlith.Service("world", worldSvc)
}

Lambda

In the “hello-world” handler Lambda, use the client methods on your service objects to call the services. Use the words returned from the service calls to construct the response to the user.

func handler(ctx context.Context, req events.APIGatewayProxyRequest) (events.APIGatewayProxyResponse, error) {
  hello, err := helloSvc.Word(ctx)
  if err != nil {
    return rest.Respond(http.StatusInternalServerError, err)
  }

  world, err := worldSvc.Word(ctx)
  if err != nil {
    return rest.Respond(http.StatusInternalServerError, err)
  }

  helloWorld := fmt.Sprintf("%s, %s!", strings.Title(hello), world)
  return rest.Respond(http.StatusOK, helloWorld)
}

Ship It

That’s all it takes to factor into services. You should be ready to deploy.

git add .
git commit -m "Factored into services"
git push origin master

Once deployed, you can make requests against your domain. You should see the same response as before.

curl https://prod.my-domain.com/

You might notice that this deploy takes longer than the last deploy. This is because Microlith uses safe Lambda deploys. By default, the test environment transitions traffic using the “AllAtOnce” configuration and the production environment uses the “Canary10Percent5Minutes” configuration. These are both configurable via the bootstrap process.
